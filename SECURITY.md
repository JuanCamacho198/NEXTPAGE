# Security Policy

## Supported Versions

We release patches for security vulnerabilities. The following versions are currently supported:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | ✅ Yes (latest)    |
| < 1.0   | ❌ No              |

## Reporting a Vulnerability

If you discover a security vulnerability in NextPage, please report it responsibly.

### How to Report

1. **Do NOT** create a public GitHub issue for security vulnerabilities
2. Email the maintainer directly (or use GitHub's private vulnerability reporting)
3. Include detailed steps to reproduce
4. Wait for acknowledgment before disclosing publicly

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Fix Release**: Based on severity (see below)

### Severity Classification

| Severity | Response Time | Release Target |
|----------|---------------|----------------|
| Critical | 24 hours      | 1-3 days       |
| High     | 3 days        | 1-2 weeks      |
| Medium   | 7 days        | 2-4 weeks      |
| Low      | Next release  | Monthly        |

## Security Best Practices

### For Users

- Keep your app updated
- Only import books from trusted sources
- Review app permissions

### For Contributors

- Never commit secrets, API keys, or credentials
- Use environment variables for configuration
- Follow secure coding practices
